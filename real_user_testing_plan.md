## 🚨 **应急预案**

### **常见问题快速解决**

#### **Problem 1: 页面无法访问**
```
可能原因：
- 前端服务未启动（localhost:3000）
- 端口被占用
- 防火墙阻止

解决步骤：
1. 前端团队检查npm start状态
2. 检查3000端口占用：netstat -an | grep 3000
3. 重启前端服务
4. 更换端口（如3001、3002）
```

#### **Problem 2: 登录失败**
```
可能原因：
- 后端认证服务异常
- 测试账户数据问题
- JWT token生成失败

解决步骤：
1. 后端团队检查认证API状态
2. 重置测试用户密码
3. 检查数据库连接
4. 查看服务器错误日志
```

#### **Problem 3: 药品搜索无结果**
```
可能原因：
- 数据库数据缺失
- 搜索API异常
- 前后端通信问题

解决步骤：
1. 后端团队确认药品数据存在
2. 测试API端点直接访问
3. 检查前端API调用代码
4. 查看网络请求日志
```
