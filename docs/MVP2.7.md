## 🎯 核心功能需求

### 1. 处方查询服务

#### 1.1 通过处方ID查询
**功能描述**：患者通过处方ID和验证码查询处方信息

**API定义**：
- 端点：`POST /api/v1/public/prescriptions/query`
- 认证：不需要（公开接口）
- 请求体：
  - prescriptionId: 处方ID
  - verificationCode: 验证码（后4位）

**业务规则**：
- 验证码为处方ID后4位
- 不返回任何患者隐私信息
- 仅返回药品清单和处方状态
- 限制查询频率（5次/分钟/IP）

**返回数据**：
```json
{
  "success": true,
  "data": {
    "prescriptionId": "TCM-2025-0001",
    "doctorName": "张医生",
    "clinicName": "中医诊所",
    "status": "paid",
    "medicines": [
      {
        "name": "当归",
        "quantity": 10,
        "unit": "克",
        "dosage": "每日一次"
      }
    ],
    "totalDoses": 7,
    "instructions": "饭后服用",
    "createdAt": "2025-01-09T10:00:00Z",
    "qrCode": "data:image/png;base64,..."
  }
}
```

**测试用例**：
- ✅ 正确验证码查询
- ✅ 错误验证码拒绝
- ✅ 频率限制验证
- ✅ 数据脱敏完整性
- ✅ 不存在的处方ID

#### 1.2 处方状态跟踪
**功能描述**：获取处方的履约状态

**API定义**：
- 端点：`GET /api/v1/public/prescriptions/:id/status`
- 认证：不需要
- 路径参数：处方ID
- 查询参数：code（验证码）

**状态信息**：
- created: 已创建
- paid: 已支付
- dispensed: 已配药
- completed: 已完成

**返回数据**：
```json
{
  "success": true,
  "data": {
    "currentStatus": "dispensed",
    "statusHistory": [
      {
        "status": "created",
        "timestamp": "2025-01-09T10:00:00Z"
      },
      {
        "status": "paid",
        "timestamp": "2025-01-09T10:05:00Z"
      },
      {
        "status": "dispensed",
        "timestamp": "2025-01-09T14:00:00Z",
        "pharmacy": "康泰药房"
      }
    ]
  }
}
```

**测试用例**：
- ✅ 状态查询准确性
- ✅ 历史记录完整性
- ✅ 药房信息显示
- ✅ 验证码验证

### 2. 药房位置服务

#### 2.1 附近药房查询
**功能描述**：基于位置查询附近的合作药房

**API定义**：
- 端点：`GET /api/v1/public/pharmacies/nearby`
- 认证：不需要
- 查询参数：
  - lat: 纬度
  - lng: 经度  
  - radius: 搜索半径（米，默认5000）
  - limit: 返回数量（默认10）

**业务规则**：
- 不存储用户位置
- 仅计算距离返回结果
- 返回营业状态和基本信息
- 支持结果缓存

**返回数据**：
```json
{
  "success": true,
  "data": [
    {
      "id": "pharmacy_1",
      "name": "康泰药房",
      "address": "奥克兰市中心XX街123号",
      "distance": 1250,
      "phone": "09-123-4567",
      "businessHours": {
        "monday": "09:00-18:00",
        "tuesday": "09:00-18:00"
      },
      "isOpen": true,
      "hasStock": true,
      "coordinates": {
        "lat": -36.8485,
        "lng": 174.7633
      }
    }
  ],
  "meta": {
    "total": 5,
    "radius": 5000
  }
}
```

**测试用例**：
- ✅ 距离计算准确性
- ✅ 营业时间判断
- ✅ 搜索半径限制
- ✅ 结果排序正确
- ✅ 坐标有效性验证

#### 2.2 药房详情查询

### 3. 匿名反馈系统

#### 3.1 提交服务评价
**功能描述**：对处方服务进行匿名评价

**API定义**：
- 端点：`POST /api/v1/public/feedbacks/service`
- 认证：不需要
- 请求体：
  - prescriptionId: 处方ID
  - rating: 评分（1-5）
  - categories: 评价维度
  - comment: 文字评价（可选）

**业务规则**：
- 基于设备ID/IP限制频率（1次/天）
- 不关联任何个人信息
- 自动过滤敏感词
- 评价数据聚合统计

**评价维度**：
- 服务态度
- 配药速度
- 药品质量
- 整体满意度

**测试用例**：
- ✅ 频率限制有效
- ✅ 匿名性验证
- ✅ 敏感词过滤
- ✅ 数据统计准确

#### 3.2 提交疗效反馈
**功能描述**：对治疗效果进行反馈

**API定义**：
- 端点：`POST /api/v1/public/feedbacks/efficacy`
- 认证：不需要
- 请求体：
  - prescriptionId: 处方ID
  - effectiveRating: 疗效评分（1-5）
  - sideEffects: 副作用描述（可选）
  - continueTreatment: 是否继续治疗

**业务规则**：
- 处方创建7天后可评价
- 同样的频率限制
- 数据用于统计分析
- 完全匿名化处理

**测试用例**：
- ✅ 时间限制验证
- ✅ 重复提交防护
- ✅ 数据脱敏处理

## 🔐 隐私保护措施

### 数据最小化
- 不收集患者姓名
- 不收集出生日期
- 不收集联系方式
- 不创建用户账户

### 访问控制
- 处方查询需要验证码
- 评价完全匿名
- 位置不做存储
- IP仅用于限流

### 日志脱敏
- 不记录查询内容
- 仅记录访问频率
- 定期清理日志
- 聚合统计数据
